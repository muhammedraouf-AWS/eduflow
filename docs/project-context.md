## 1. Project Overview

**EduFlow** is a production-grade Learning Management System (LMS) inspired by Udemy. Instructors create and sell rich video courses; students browse, enroll, watch, track progress, and earn certificates.

**Personas**

- **Student** — browses, enrolls, watches lessons, tracks progress, leaves reviews.
- **Instructor** — creates/edits courses, uploads videos & resources to Cloudinary, sees analytics.
- **Admin** — moderates content, manages users, sees platform-wide analytics.

**Non-goals (for v1)** — live classes, real-time chat, mobile apps, multi-tenancy.

## 3. Architecture Decisions

### 3.1 Why App Router (not Pages Router)

React Server Components, streaming, server actions, co-located layouts with route-level auth guards. Drastically reduces client JS for content-heavy pages (catalog, course detail).

### 3.2 Why feature-based architecture

A type-based layout breaks down past ~20 features. Feature folders bundle UI + actions + queries + validations + types together, giving high cohesion and easy deletion/extraction. Cross-feature primitives live under `src/components/ui` and `src/components/shared`.

### 3.3 Server Actions over REST API routes (default)

Use **server actions** for mutations called from our own React tree. Reserve **`/app/api/*`** for:

- Webhook receivers (Stripe, Cloudinary)
- Public endpoints (signed-upload signatures)
- Anything called by a non-Next client (mobile app later)

### 3.4 Database sessions over JWT (Auth.js)

- Role changes (student → instructor) take effect immediately.
- Sessions can be revoked on demand (admin ban, password change).
- Required for moderation features in Phase 5.

### 3.5 Signed direct uploads to Cloudinary (never proxy)

Browser uploads straight to Cloudinary using a short-lived signature generated by a server action. The Vercel function never sees the file bytes — saves memory, time, and bandwidth costs.

### 3.6 Defense in depth

Three independent enforcement layers for every protected action:

1. **Middleware** — gates entire route segments.
2. **Server (action / route handler)** — re-checks session + role + ownership on every call.
3. **UI** — conditional rendering for nice UX (NOT a security boundary).

### 3.7 Type-safe environment variables

`src/lib/env.ts` validates every env var with Zod at build & runtime. The app refuses to start if anything is missing or malformed — catches misconfigured Vercel deployments before they ship.

### 3.8 Decimal for money, CUIDs for IDs

- `Prisma.Decimal` for course prices (no float rounding errors).
- CUIDs (URL-safe, sortable) for primary keys.
- Soft deletes via status enums (e.g., `CourseStatus.ARCHIVED`) — never lose enrolled students' history.

## 8. Coding Conventions

- **TypeScript strict mode** + `noUncheckedIndexedAccess` + `noImplicitOverride`.
- **No default exports** for components except Next.js route files (`page.tsx`, `layout.tsx`, etc.).
- **Server Components by default**; opt into client only when interactivity / hooks are needed.
- **Zod schemas live in `features/*/validations/`** and are the contract shared by RHF + server action.
- **Re-validate server-side** on every mutation — never trust the client.
- **Imports order** (Prettier-friendly): builtin → external → `@/...` → relative.
- **Component naming**: `PascalCase.tsx` for components, `kebab-case.ts` for everything else.
- **Forms**: React Hook Form + `@hookform/resolvers/zod` + `<FormField />` shadcn pattern.
- **Errors**: throw in actions, render with error boundaries; show user-friendly toast via Sonner.

## 9. Reusable Patterns

- **`db` singleton** (`src/lib/db.ts`) — survives HMR, never spawns connection storms.
- **`env` object** (`src/lib/env.ts`) — single typed source for all env vars.
- **Cloudinary folder map** (`cloudinaryFolders.*`) — one helper per resource type, no string-building in callers.
- **`cn()` helper** (`src/lib/utils.ts`) — `clsx` + `tailwind-merge` for conditional class names.
- **`AppProviders`** (`src/providers/index.tsx`) — single client provider tree; new providers added once.
- **`siteConfig`** — single source for app name, description, URLs used by metadata + UI.

## 15. Important Decisions Log

- **2026-05-07** — Chose **npm** as package manager (user preference; widest compatibility).
- **2026-05-07** — Chose **Neon** for Postgres (serverless, free tier, zero setup, Vercel-friendly).
- **2026-05-07** — Public GitHub repo, manually created by user.
- **2026-05-07** — Chose **Auth.js database sessions** over JWT for instant role updates and revocation support.
- **2026-05-07** — Chose **`prisma-client` generator** (Prisma 7) outputting to `src/generated/prisma` (gitignored, regenerated on each `db:generate`).
- **2026-05-07** — Set `noUncheckedIndexedAccess: true` — array/object access returns `T | undefined`. Slightly more verbose, catches a class of real bugs (especially around dynamic course/chapter lookups).
- **2026-05-07** — **Prisma 7 driver adapters are mandatory.** Installed `@prisma/adapter-pg` (uses node-postgres). Compatible with Neon's pooled URL, local Postgres, RDS, etc. Future upgrade path: `@prisma/adapter-neon` for Edge Runtime + Neon serverless WebSocket driver if we need Edge auth later.
- **2026-05-07** — shadcn `base-nova` preset uses **Base UI** (not Radix). Different mental model: instead of `asChild`, we pass `render={<a />}` and (for non-button elements) `nativeButton={false}`. Documented so future contributors don't trip on this.
- **2026-05-07** — `src/generated/prisma` is **gitignored** and must be regenerated after every clone (`npm run db:generate`). This keeps the repo small and avoids stale checked-in clients.
